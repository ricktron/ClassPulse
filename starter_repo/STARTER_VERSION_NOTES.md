# Starter Version Notes

## Current version
- v1

## Status
- Usable internal trial
- Ready to use for new repos or student starts
- Expected to improve through real use, not abstract planning alone

## What this version includes
- A compact human front door in `START_HERE.md`
- A simple startup intake in `STARTER_CARD.md`
- A short AI kickoff file in `START_HERE_FOR_LLM.md`
- A simplified operating spine based on Project Forge patterns
- Core repo guidance files such as `AGENTS.md`, `DECISIONS.md`, and `docs/PROJECT_STATUS.md`
- A lighter student-usable path supported by `docs/bootstrap/STUDENT_MODE.md` in Forge

## How to use this starter
1. Start with `START_HERE.md`.
2. Fill out `STARTER_CARD.md`.
3. If using AI, upload `START_HERE_FOR_LLM.md` and the filled starter card in your first message.
4. Begin building.
5. Keep the core files unless you understand why you are changing or removing them.

## Core files in this version

- `START_HERE_FOR_LLM.md`

- `docs/ai/AGENT_STARTUP_CHECKLIST.md`

## What this version does not include
- A generator
- Upgrade tooling
- A repo doctor or automatic repo audit
- Automatic stack detection
- Main Project Forge repo integration beyond the copied starter structure
- Coverage for every specialized repo type

## Known limitations
- Best suited for general coding repos, not every possible project
- Still tuned more for practical use than for public distribution
- May need small adjustments for specialized stacks or unusual workflows
- Best judged by real use with new projects or students

## Candidate next improvements
- A small discoverability link from Forge root docs to the bootstrap area
- An optional sanity-check runbook stub once a real stack lands
- A later template-repo distribution path if the starter proves itself
- Further refinement after one or two real student or personal uses

## Change policy
- Adapt this starter to fit your project.
- Do not silently delete or heavily compress meaningful guidance without understanding its role first.
- Prefer small, clear changes over large rewrites.
- Let real use drive future changes.

## Source / lineage
Derived from Project Forge operating patterns. Preserve process spine, not Project Forge domain-specific product scope. Adapt freely for your own project or classroom use.
